# Momoapi 平台文档

> Momoapi 平台提供多模型 LLM API 中转服务,支持 Claude、GPT、Grok 等模型

## 欢迎使用 Momoapi 平台

Momoapi 平台是一个 **多模型 LLM API 中转服务**。你只需一个 Token,就能在同一地址调用 Claude、GPT、Grok 等主流模型,用于 Claude Code、Codex 等 AI 编程工具,或直接通过 API 调用。

## 核心能力

- **统一 API 中转**: 一个 Base URL \+ 一个 Token,调用所有模型。兼容 OpenAI / Anthropic / xAI 协议。
- **丰富的模型**: Grok 全系列、GPT-5.x、Claude 全系列,以及图片/视频生成模型。
- **客户端接入**: 一键配置 Claude Code、Codex 等主流 AI 编程客户端。
- **账户与计费**: 注册账户、管理 Token、查看用量与余额。

## 适用场景

Momoapi 适合需要直连访问、多模型切换、统一 Token 管理和多客户端复用的用户。

核心能力：

| 能力 | 说明 |
| --- | --- |
| 全球直连 | 无需额外网络环境即可访问中转服务 |
| 一个 Key 复用 | 同一个账号下可创建不同用途的 API Key |
| 多模型支持 | 可按控制台实际权限选择 Claude、GPT、Grok、GLM、图片与视频等模型 |
| 额度管理 | API Key 可设置额度、有效期和使用范围 |
| 客户端兼容 | 支持 Claude Code、Codex、OpenClaw、OpenAI SDK 等工具 |

## 快速开始

只需三步即可开始使用:

1. **注册并获取 Token**: 在 [Momoapi 控制台](https://momoapi.top/console) 注册账号,创建一个以 `sk-` 开头的 API Token。详见 [获取 Token](/getting-token)。
2. **选择要用的模型**: 查看 [模型与定价](/models/overview),了解每个模型的特点和价格,选一个适合你的。
3. **配置到客户端**: 把 Token 配置到 Claude Code、Codex 等客户端。详见 [客户端配置](/cli-config/env-check)。

## 关键信息

在使用 Token 时,你需要以下信息:

| 参数 | 值 |
| --- | --- |
| **Base URL** | `https://momoapi.top` |
| **API Key** | 你创建的 Token(`sk-xxx`) |

> **提示**: Base URL 不带 `/v1`,客户端会自动补全。
