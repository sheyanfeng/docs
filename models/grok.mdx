# Grok 系列模型

> xAI Grok 图片与视频生成模型的使用说明与选型指南

Grok 是 xAI 推出的多模态生成模型系列。Momoapi 当前已上架 Grok 图片与视频生成模型,模型名以前缀区分不同上游与版本,调用时需要完整填写模型 ID。

## 图片模型详解

### `ls-/grok-4.2-image`

Grok 4.2 图片生成模型,适合需要更高画面质量、细节稳定性和成片质感的任务。

- **适合**:海报、封面、产品图、角色概念图、需要更细节的创意图片
- **计费**:按次计费,每次 \$0.192

### `ls-/grok-4.1-image`

Grok 4.1 图片生成模型,成本更低,适合常规文生图与批量试图。

- **适合**:日常配图、草图验证、批量生成、成本敏感的图片任务
- **计费**:按次计费,每次 \$0.160

## 视频模型详解

### `dy-/grok-video-3`

Grok Video 3 标准视频模型,适合通用文生视频场景。没有明确固定时长需求时,优先使用该模型。

- **适合**:通用短视频、动态镜头、产品展示、社媒素材
- **计费**:按次计费,每次 \$0.520

### `dy-/grok-video-3-10s` / `dy-/grok-video-3-15s`

Grok Video 3 的固定时长版本,模型名中已经标明目标时长。需要 10 秒或 15 秒成片时,优先选择对应模型。

- **适合**:固定时长广告片、口播转场、短视频片段
- **计费**:按次计费,每次 \$0.520

### `dy-/grok-1.5-video-6s` / `dy-/grok-1.5-video-10s` / `dy-/grok-1.5-video-15s`

Grok 1.5 视频系列,按 6 秒、10 秒、15 秒拆分成不同模型。适合按固定时长快速出片。

- **适合**:短镜头测试、批量视频素材、固定时长创意片段
- **计费**:按次计费,每次 \$0.520

### `ls-/grok-video-3`

Grok Video 3 高质量线路,价格更高,适合对画面稳定性和最终效果要求更高的视频任务。

- **适合**:重要成片、商业素材、需要更高质量的视频生成
- **计费**:按次计费,每次 \$0.800

## 调用示例

### 图片生成

```bash
curl https://momoapi.top/v1/images/generations \
  -H "Authorization: Bearer sk-你的Token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "ls-/grok-4.2-image",
    "prompt": "A futuristic sports car parked on a neon city street, cinematic lighting",
    "n": 1,
    "response_format": "url"
  }'
```

### 视频生成

```bash
curl -X POST https://momoapi.top/v1/video/generations \
  -H "Authorization: Bearer sk-你的Token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "dy-/grok-video-3-10s",
    "prompt": "A golden retriever running across a beach at sunset, handheld cinematic shot"
  }'
```

返回任务信息后,记录 `task_id`,再轮询任务状态直到生成完成。

```bash
curl https://momoapi.top/v1/video/generations/{task_id} \
  -H "Authorization: Bearer sk-你的Token"
```

<Warning>
  图片和视频结果链接通常是临时链接,生成完成后请尽快下载保存。
</Warning>

## 参数建议

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `model` | string | 是 | 填写完整模型名,例如 `ls-/grok-4.2-image` 或 `dy-/grok-video-3-10s` |
| `prompt` | string | 是 | 描述要生成的画面。图片写清主体、风格、构图、光线;视频写清动作、镜头、运镜、氛围 |
| `n` | int | 否 | 图片生成数量。每张单独计费,视频不支持批量 |
| `response_format` | string | 否 | 图片可返回 `url` 或 `b64_json`,默认推荐 `url` |

## 选型建议

| 场景 | 推荐 |
| --- | --- |
| 图片质量优先 | `ls-/grok-4.2-image` |
| 图片成本优先 | `ls-/grok-4.1-image` |
| 通用视频生成 | `dy-/grok-video-3` |
| 10 秒视频 | `dy-/grok-video-3-10s` 或 `dy-/grok-1.5-video-10s` |
| 15 秒视频 | `dy-/grok-video-3-15s` 或 `dy-/grok-1.5-video-15s` |
| 高质量视频线路 | `ls-/grok-video-3` |

## 常见问题

<Accordion>
  <AccordionItem title="模型名前面的 dy-/ 和 ls-/ 要保留吗?">
  要保留。`dy-/` 和 `ls-/` 是模型 ID 的一部分,调用接口时需要完整填写,否则可能找不到对应模型。
</AccordionItem>

  <AccordionItem title="视频模型还需要传 seconds 吗?">
  固定时长模型已经在模型名中标明 6s、10s 或 15s,通常直接选择对应模型即可。具体可用参数以控制台和接口返回为准。
</AccordionItem>

  <AccordionItem title="怎么估算费用?">
  本页模型均按次计费。费用 = 单次价格 × 调用次数 × 你所在分组的折扣倍率,最终以控制台账单为准。
</AccordionItem>
</Accordion>
