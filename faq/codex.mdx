### Codex 连接失败

**症状**：运行 `codex` 后无法获取回复，或请求直接报错。

先检查环境变量是否正确：

```bash
echo $OPENAI_BASE_URL
# 应显示: https://momoapi.top/v1

echo $OPENAI_API_KEY
# 应显示你的令牌
```

注意：Codex 使用 OpenAI 兼容接口，`OPENAI_BASE_URL` 必须包含 `/v1` 后缀。

还需要确认当前 API Key 所属分组包含 GPT / OpenAI 兼容模型。

### 模型不可用

如果提示某个模型不可用，通常是令牌分组没有包含该模型。处理方式：

1. 到控制台检查当前令牌可用模型。
2. 切换到可用模型后重试。
3. 如需固定使用某个模型，可创建包含该模型的新令牌。

### 环境变量未生效

修改 `~/.zshrc` 或 `~/.bashrc` 后，需要让配置重新加载：

```bash
source ~/.zshrc
# 或
source ~/.bashrc
```

也可以直接关闭并重新打开终端。

Windows 用户修改系统环境变量后，也需要重新打开终端窗口；如果仍不生效，建议重新启动应用或系统后再试。
