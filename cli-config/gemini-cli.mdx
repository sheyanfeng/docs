# Gemini CLI 配置使用教程

Gemini CLI 是 Google 提供的命令行 AI 工具，可以直接在终端里读取项目、解释代码、生成文件、执行脚本和处理自动化任务。

本页介绍如何安装 Gemini CLI，并通过 MomoAPI 配置中转地址和 API Key。

## 一、准备工作

开始前请先准备：

- Node.js 20 或更高版本
- Windows、macOS 或 Linux 终端
- 一个 MomoAPI Token，通常以 `sk-` 开头

如果还没有 Token，请先进入 [MomoAPI 控制台](http://momoapi.top) 创建。

## 二、安装 Gemini CLI

### 方式一：使用 npx 即时运行

```bash
npx https://github.com/google-gemini/gemini-cli
```

### 方式二：使用 npm 全局安装

```bash
npm install -g @google/gemini-cli
```

安装完成后，可以执行：

```bash
gemini --version
```

如果能看到版本号，说明安装成功。

## 三、推荐方式：使用配置文件

Gemini CLI 的全局配置目录是用户目录下的 `.gemini` 文件夹。

| 系统 | 配置目录 |
| --- | --- |
| Windows | `C:\Users\<你的用户名>\.gemini` |
| macOS / Linux | `~/.gemini` |

如果目录不存在，请先手动创建。

### 第一步：创建 .env

在 `.gemini` 目录下创建 `.env` 文件，写入以下内容：

```bash
GOOGLE_GEMINI_BASE_URL=http://momoapi.top
GEMINI_API_KEY=sk-你的Token
GEMINI_MODEL=gemini-3-pro-preview
```

参数说明：

| 参数 | 说明 |
| --- | --- |
| `GOOGLE_GEMINI_BASE_URL` | MomoAPI 的 Gemini CLI 中转地址，填写 `http://momoapi.top` |
| `GEMINI_API_KEY` | 你的 MomoAPI Token，替换成 `sk-` 开头的真实 Key |
| `GEMINI_MODEL` | 默认模型，可按控制台可用模型调整 |

<Warning>
  `GEMINI_API_KEY` 不要写成示例值。请把 `sk-你的Token` 替换成你自己在 MomoAPI 创建的 Token。
</Warning>

### 第二步：创建 settings.json

在 `.gemini` 目录下创建 `settings.json`，写入：

```json
{
  "ide": {
    "enabled": true
  },
  "security": {
    "auth": {
      "selectedType": "gemini-api-key"
    }
  }
}
```

保存后，重新打开终端，让 Gemini CLI 读取新配置。

## 四、可选方式：设置环境变量

如果不想使用 `.env` 文件，也可以直接设置系统环境变量。

### macOS / Linux

```bash
export GOOGLE_GEMINI_BASE_URL=http://momoapi.top
export GEMINI_API_KEY=sk-你的Token
export GEMINI_MODEL="gemini-3-pro-preview"
```

如果希望每次打开终端自动生效，可以把这些内容加入 `.bashrc`、`.zshrc` 或其它 shell 配置文件。

### Windows PowerShell

```powershell
setx GOOGLE_GEMINI_BASE_URL "http://momoapi.top"
setx GEMINI_API_KEY "sk-你的Token"
setx GEMINI_MODEL "gemini-3-pro-preview"
```

执行 `setx` 后，请关闭当前 PowerShell 窗口并重新打开。

## 五、快速开始

在当前项目目录启动：

```bash
gemini
```

指定多个目录：

```bash
gemini --include-directories ../lib,../docs
```

临时指定模型：

```bash
gemini -m gemini-3-pro-preview
```

非交互模式发送提示词：

```bash
gemini -p "解释这个代码库的架构"
```

输出 JSON：

```bash
gemini -p "解释这个代码库的架构" --output-format json
```

## 六、常见使用场景

- 让 Gemini CLI 解释项目结构和关键模块。
- 根据需求生成新文件或修改现有代码。
- 让它总结最近的代码改动。
- 使用自然语言排查报错原因。
- 通过 MCP 扩展更多工具能力。
- 使用 `GEMINI.md` 为项目补充长期上下文。

## 七、常见问题

### 命令无法识别

请确认 Node.js 版本不低于 20，并确认 `npm install -g @google/gemini-cli` 已执行成功。Windows 用户如果刚安装完成，请重新打开终端。

### 鉴权失败

请检查：

1. `GEMINI_API_KEY` 是否已替换为真实 MomoAPI Token。
2. Token 是否仍然有效。
3. `GOOGLE_GEMINI_BASE_URL` 是否为 `http://momoapi.top`。
4. `.env` 是否放在正确的 `.gemini` 目录下。

### 模型不可用

请把 `GEMINI_MODEL` 改成 MomoAPI 控制台当前支持的 Gemini 模型，然后重新打开终端再试。

### 配置修改后没有生效

请关闭当前终端窗口并重新打开。如果使用的是环境变量方式，Windows 用户需要重新打开 PowerShell；macOS / Linux 用户可以重新加载 shell 配置文件。
